## ⚠️ 代码修改纪律（强制执行）

本规则适用于所有代码修改，任何修改前必须逐条检查。

### 核心4条

1. **先想再写** — 说清假设，不确定就问，别猜。修改前先陈述：问题出在哪、打算改什么、为什么。
2. **简单优先** — 用最少代码解决问题。能改一行的不要重构整个文件。不引入当前不需要的设计模式。
3. **外科手术式修改** — 只动需要改的地方。不改变周围代码的缩进、格式、命名。不移动、删除、重写不在本次任务范围内的代码。**绝对不碰没有被要求修改的功能。**
4. **目标驱动执行** — 定义成功标准，验证通过才算完成。改完后说明改了什么、预期效果是什么。

### 扩展8条

5. **模型只做判断** — 路由、重试等确定性逻辑不要用LLM，用代码写死。
6. **硬Token预算** — 如果一个修改越改越复杂，停下来重新评估方案。不要陷入"改A坏了B，修B又坏了C"的连锁反应。
7. **冲突选一边** — 一个文件里不要同时存在两种实现方案。选定方向，彻底执行。
8. **先读再写** — 修改任何文件前，先读它的当前完整内容。了解这个文件被谁引用、引用了谁。了解修改会影响哪些其他页面或组件。
9. **测试验证意图** — 验证用户要求的效果，不是验证自己写的代码能跑。
10. **每步检查点** — 每步做完总结：做了什么、验证了什么、还剩什么。不要一口气改十个文件然后说"全部完成"。
11. **遵循代码库惯例** — 沿用项目现有的命名规范、文件结构、CSS方法论。不引入项目中没有的新依赖，除非被明确要求。
12. **失败要大声** — 如果改完后不确定是否正确，明确说出来。不要用"应该没问题"掩盖不确定性。遇到无法解决的问题，立刻上报而不是绕过。

### 执行检查清单（每次修改前过一遍）

1. 我读了要改的文件的当前内容吗？
2. 我清楚这次修改的范围吗？（只改X，不碰Y和Z）
3. 我能用最少的改动解决问题吗？
4. 改完后我怎么验证？
5. 有没有可能影响到其他页面/组件？

## 全局规范

### 字体
- 品牌名 + 英文标题 + 导航：本地字体 HelveticaNeue93BlackExt（文件在 assets/fonts/）
- 中文标题：微软雅黑 Bold（系统字体，CSS直接调用 font-family: "Microsoft YaHei"）
- 中文正文：微软雅黑 Regular
- 英文正文/标签：HelveticaNeue93BlackExt（小字号时）
- 详细字体参数见 docs/字体规范.md

### @font-face 统一声明（全站只声明一次，写在 global.css 中）
```css
@font-face {
  font-family: 'HelveticaNeue93BlackExt';
  src: url('../assets/fonts/HelveticaNeue93BlackExt.otf') format('opentype');
  font-weight: 100 900;
  font-style: normal;
  font-display: swap;
}
```
所有页面引用同一份 global.css，不要在各页面CSS文件里重复声明 @font-face。

### 颜色（全站统一色板）
```css
:root {
  /* 品牌色 */
  --brand-blue: #146EF5;

  /* 背景 */
  --bg-primary: #F4F4F5;

  /* 文字 */
  --text-primary: #1c1c1e;
  --text-secondary: rgba(0,0,0,0.4);
  --text-nav: #000000;

  /* 品牌元素 */
  --brand-logo: #146EF5;
  --brand-title: #146EF5;
}
```

**颜色使用规则：**
- 品牌 logo（所有页面导航栏左侧 "maCher Chan®"）：`#146EF5` 蓝
- 首页大字 "MACHER CHAN"：`#146EF5` 蓝
- 首页副标题 "DESIGN & TREND STUDIO"：`#146EF5` 蓝
- 导航栏链接（PROJECTS / STUDIO / SERVICES / CONTACT）：`#000000` 黑
- 导航栏当前页下划线：`#000000` 黑
- 正文内容文字：`#1c1c1e` 深灰
- 次级文字（标签、小字、元数据）：`rgba(0,0,0,0.4)`
- 所有页面背景：`#F4F4F5`（包括 Studio 页）
- 图片容器背景：`#F4F4F5`（和页面背景同色）
- 可点击链接/按钮：`#146EF5` 蓝
- 品牌强调色（少量使用）：`#FF2D1A` 红 / `#00C853` 绿 / `#FFE500` 黄

### 导航栏（所有页面统一）
- 固定在顶部
- 左侧：maCher Chan®（品牌字体，`#146EF5` 蓝，点击回首页）
- 右侧：PROJECTS / STUDIO / SERVICES / CONTACT（品牌字体，14px，`#000000` 黑）
- 当前页链接加下划线（黑色）
- 导航栏背景透明，滚动时不变色
- hover效果：下划线出现
- 不要汉堡菜单，直接展示所有链接

### 图片显示规则（全站统一）
- 所有look图片容器统一为3:4比例（宽:高）
- 图片使用 object-fit: contain
- 容器背景色 `#F4F4F5`
- 绝对不使用 object-fit: cover 裁切图片
- 图片容器不加 border、box-shadow、outline

### 无数字标注
- 全站不使用"6 SERIES"、"3 PACKAGES"等数字标注
- 不使用项目编号（01/07之类）

### 散落全览视图（默认视图）

**布局：**
- 背景 `#F4F4F5`
- 展示该系列全部look图片，占满100vw
- 每行最多2-3张，大小比不超过2:1

## 技术实现

### 技术栈
- HTML5 + CSS3 + Vanilla JS
- GSAP（可选，从CDN引入）
- 不使用React/Vue/任何框架
- 不使用Three.js/WebGL

### 文件结构
```
macher-website/
├── index.html
├── projects.html
├── project-detail.html
├── studio.html
├── services.html
├── contact.html
├── css/
│   ├── global.css          (唯一的 @font-face + CSS变量 + 导航栏 + reset)
│   └── [页面].css
├── js/
│   └── [页面].js
├── assets/
│   ├── fonts/              (HelveticaNeue93BlackExt.otf)
│   └── projects/
└── docs/
    ├── references/
    └── *.md
```

**global.css 是唯一声明 @font-face 和 CSS 变量的地方。各页面CSS不要重复声明。所有页面 head 中先引入 global.css。**
